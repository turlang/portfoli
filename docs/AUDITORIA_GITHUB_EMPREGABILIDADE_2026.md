# Auditoria GitHub — Empregabilidade 2026

Este documento organiza os repositórios do GitHub `turlang` com foco em empregabilidade para vagas de Desenvolvedor Full Stack, Front-end, Back-end Node.js e projetos SaaS.

## Objetivo

Fazer o GitHub funcionar como uma vitrine profissional, permitindo que recrutadores encontrem rapidamente os projetos mais fortes, entendam as tecnologias utilizadas e percebam capacidade de entrega de produtos reais.

## Projetos que devem aparecer em destaque

### 1. GlossFlow / glossflow1

**Posicionamento:** SaaS completo para salões de beleza, barbearias e clínicas de estética.

**Por que fortalece empregabilidade:**

- Demonstra arquitetura SaaS.
- Usa React, Vite, Node.js, Fastify, TypeScript, Prisma e MongoDB.
- Possui módulos de negócio reais: agendamento, CRM, estoque, financeiro, comissões, fidelidade, automações e multiempresa.
- Mostra capacidade de construir produto com painel administrativo, autenticação e regras de negócio.

**Ação recomendada:**

- Usar `glossflow1` como versão principal.
- Padronizar README com screenshots, arquitetura e link de demonstração.
- Evitar deixar `glossflow` e `glossflow1` competindo sem explicação. Um deve ser versão principal e o outro pode ser legado.

---

### 2. Prospe--o-clientes / LeadHunter Pro

**Posicionamento:** SaaS de prospecção comercial, geração de leads, CRM e assinaturas.

**Por que fortalece empregabilidade:**

- Demonstra backend com Express, MongoDB, JWT, Mongoose e Mercado Pago.
- Tem funcionalidades comerciais claras: cadastro, login, CRM, pipeline, exportação CSV, planos, painel admin e controle de assinaturas.
- Excelente para mostrar capacidade de transformar problema de negócio em produto.

**Ação recomendada:**

- Padronizar nome público como `LeadHunter Pro`.
- Criar README com fluxo do usuário, plano gratuito, planos pagos e prints do CRM.
- Destacar Mercado Pago e segurança anti-abuso.

---

### 3. SaaS Mestre Orc

**Posicionamento:** Plataforma para mestres de RPG gerenciarem campanhas, jogadores, reservas e conteúdo gerado.

**Por que fortalece empregabilidade:**

- Mostra criatividade, domínio de produto e construção de dashboards.
- Tem integração conceitual com IA, Foundry VTT, marketplace e exportação de conteúdo.
- Diferencia o perfil, pois foge de projetos comuns de CRUD genérico.

**Ação recomendada:**

- Posicionar como projeto autoral de produto digital.
- Deixar claro o status técnico: protótipo avançado, front-end funcional ou SaaS em evolução.
- Conectar com o projeto `Mestre-orc` como ferramenta auxiliar.

---

### 4. Papel Parede

**Posicionamento:** Landing page React para venda de papéis de parede e decoração.

**Por que fortalece empregabilidade:**

- Demonstra React, componentização, UX/UI, responsividade, catálogo e chamada para WhatsApp.
- É bom para vagas front-end e trabalhos freelancer.

**Ação recomendada:**

- Usar como case de conversão para negócio local.
- Inserir screenshots e link online.
- Destacar foco em CTA, catálogo e experiência mobile.

---

### 5. Orçamento Pessoal

**Posicionamento:** Dashboard financeiro com PWA.

**Por que fortalece empregabilidade:**

- Mostra manipulação de dados, interface de dashboard, formulários e recursos PWA.
- Bom para provar lógica de negócio e experiência com dados financeiros.

**Ação recomendada:**

- Melhorar README com prints do dashboard.
- Destacar service worker, manifesto e armazenamento local.

Projetos simples são bons para aprendizado, mas podem reduzir a percepção de senioridade se aparecerem com o mesmo peso dos projetos principais.

- `calculadora`
- `cronometro`
- `conversor-simples-de-moeda`
- `sorteio-radom`
- `Jokenp-`

Caso permaneçam públicos, coloque no README: **Projeto de estudo inicial**.

## Problemas de apresentação encontrados

1. Existem projetos duplicados ou versões parecidas, como `glossflow` e `glossflow1`.
2. Alguns nomes de repositórios têm grafia inconsistente ou caracteres quebrados.
3. Projetos fortes ficam misturados com estudos simples.
4. Nem todos os READMEs seguem o mesmo padrão profissional.
5. Faltam screenshots, GIFs e links de deploy destacados nos READMEs.
6. Falta um repositório especial `turlang/turlang` para o README do perfil principal.
