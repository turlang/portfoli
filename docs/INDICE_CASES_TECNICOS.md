# Índice dos Cases Técnicos

Este índice organiza os principais projetos do GitHub como estudos de caso voltados para empregabilidade.

## Projetos principais

### 1. GlossFlow Enterprise

- Repositório: `turlang/glossflow1`
- Documento: `docs/CASE_TECNICO.md`
- Tipo: SaaS Full Stack
- Destaque: React, Node.js, Fastify, TypeScript, Prisma, MongoDB, CRM, agenda e multiempresa.

### 2. LeadHunter Pro

- Repositório: `turlang/Prospe--o-clientes`
- Documento: `docs/CASE_TECNICO.md`
- Tipo: SaaS de prospecção e CRM
- Destaque: Express, MongoDB, JWT, Mercado Pago, planos, CRM e painel administrativo.

### 3. SaaS Mestre Orc

- Repositório: `turlang/saas-mestre-orc`
- Documento: `docs/CASE_TECNICO.md`
- Tipo: Produto autoral / SaaS nichado
- Destaque: Dashboard, campanhas, reservas, exportação JSON, Foundry Bridge e geração de conteúdo.

### 4. Site Papel de Parede

- Repositório: `turlang/papel-parede`
- Documento: `docs/CASE_TECNICO.md`
- Tipo: Landing page comercial
- Destaque: React, Vite, catálogo, filtros, WhatsApp CTA, UX/UI e responsividade.

### 5. FinançasPro BI

- Repositório: `turlang/orcamento-pessoal`
- Documento: `docs/CASE_TECNICO.md`
- Tipo: Dashboard financeiro/PWA
- Destaque: JavaScript, formulários, indicadores, PWA, Service Worker e responsividade.

## Como usar estes cases em entrevistas

Em entrevistas, cada projeto deve ser explicado com a seguinte estrutura:

1. Qual problema o projeto resolve.
2. Para quem ele foi feito.
3. Quais funcionalidades principais existem.
4. Qual stack foi usada.
5. Quais decisões técnicas foram tomadas.
6. Quais melhorias futuras seriam implementadas.
7. O que o projeto prova sobre sua capacidade técnica.

## Ordem recomendada para apresentação

1. GlossFlow Enterprise.
2. LeadHunter Pro.
3. SaaS Mestre Orc.
4. Site Papel de Parede.
5. FinançasPro BI.

## Mensagem profissional central

Estes projetos mostram capacidade de construir aplicações web completas, com foco em produto, usabilidade, regra de negócio, documentação e evolução técnica.
